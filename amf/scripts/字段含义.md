Event_Name 里的四个事件是脚本拆出来的四段耗时：

AMF_BCF_Register_E2E
AMF 向 BCF 注册 NF profile 的端到端耗时。从 AMF 打印 Send HTTP message to .../nbcf_management/.../nf_instances/... 开始，到 [AMF][BCF-REG] Registration SUCCESS 结束。

AMF_BCF_Auth_Init
BCF 认证第一步耗时。AMF 调用 /nbcf_auth/v1/auth/init，BCF 返回 challenge/session_id。对应日志是 HTTP POST .../auth/init 到 HTTP 200 from .../auth/init。

AMF_BCF_Auth_Verify
BCF 认证第二步耗时。AMF 对 challenge 签名后调用 /nbcf_auth/v1/auth/verify，BCF 返回 token。对应日志是 HTTP POST .../auth/verify 到 HTTP 200 from .../auth/verify。

AMF_BCF_Auth_E2E
完整认证链路耗时。从 auth/init 开始，到 AMF 状态变成 TOKEN_READY 结束。它包含 auth/init、本地签名处理、auth/verify、token 缓存和状态切换这些步骤。

所以简单说：第一个是“注册耗时”，第二第三是“认证两个 HTTP 子步骤”，第四个是“完整拿到可用 token 的认证总耗时”。
